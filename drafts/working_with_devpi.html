Test your python package with devpi

Sat 28 November 2015

devpi ia a PyPI-compatible server, it can save a lot of time developing python packages.

Scenario

You have ProjectA that depends on PackageB, and you need to patch/upgrade PackageB. So you start coding, run tests and when you are ready to release PackageB, you want to test it as dependency in ProjectA. Now you can:

  • import PackageB with pip install PackageB.tar.gz in ProjectB virtualenv

  • make a develop install pip install -e <PackageB source directory>

This works pretty well if you need to run the tests of ProjectA into the development virtualenv, but what if you want to test ProjectA using tox?

Tox will create a new virtualenv for each configuration and will look for dependencies in pypi.wfp.org or any other configured index server, it does not know anything about your local PackageB.tar.gz. Of course
